# Product

## Register

product

## Users

Analystes PIA et décideurs gouvernementaux togolais. Les analystes explorent les données par filière (production, rendement, climat, prix) pour le suivi quotidien. Les décideurs consultent les KPIs et prévisions pour le pilotage stratégique. Usage sur desktop (majoritairement), parfois tablette en réunion.

## Product Purpose

AgriDash Togo est un tableau de bord prédictif pour le pilotage des filières agricoles au Togo. Il centralise les données FAO/Banque Mondiale, climat NASA POWER, et modèles ML (Prophet, Random Forest) pour offrir : production par culture, analyse macro-économique, cartographie régionale, prévisions 5 ans, prix marché, et scoring de risque chaîne d'approvisionnement.

## Brand Personality

Data-sérieux, transparent, professionnel. Pas de "buzz" ni décoratif — chaque pixel sert l'information. Trois mots : *précis, sobre, fiable*.

## Anti-references

- Pas de glassmorphisme, gradients ou effets décoratifs
- Pas de couleurs vives / saturées — palette professionnelle
- Pas de blocs surdimensionnés — data-dense mais lisible
- Pas de "AI slop" — chartes propres, grilles claires, typo sobre
- Pas de pages de chargement chorégraphiées — on charge en tâche

## Design Principles

1. **Data first** — chaque élément de design existe pour faire ressortir l'information, pas pour décorer
2. **Familiarité outil** — le dashboard doit disparaître dans la tâche, pas attirer l'attention sur son design
3. **Hiérarchie constante** — mêmes patterns visuels partout (cartes, boutons, tableaux, graphiques)
4. **Densité maîtrisée** — beaucoup d'info mais pas de bruit : espacement cohérent, pas de surencombrement
5. **Accessible sobrement** — contrastes WCAG, animations réduites optionnelles, rien qui gêne l'usage

## Accessibility & Inclusion

WCAG AA visé. Contraste ≥ 4.5:1 texte principal, ≥ 3:1 grands textes. Mode sombre disponible. Animations limitées (< 250ms, décoratives interdites).
